### 2. 令牌桶算法实现

```typescript
private async checkQps(
  ruleId: string,
  qpsLimit: number,
  burstSize: number,
): Promise<RateLimitResult> {
  const now = new Date();
  const bucket = this.tokenBuckets.get(ruleId) || {
    tokens: burstSize,
    lastUpdate: now,
  };

  // 计算令牌补充
  const timePassed = (now.getTime() - bucket.lastUpdate.getTime()) / 1000;
  const tokensToAdd = timePassed * qpsLimit;
  bucket.tokens = Math.min(bucket.tokens + tokensToAdd, burstSize);
  bucket.lastUpdate = now;

  // 检查是否有令牌
  if (bucket.tokens < 1) {
    const waitTime = Math.ceil((1 - bucket.tokens) / qpsLimit);
    return {
      allowed: false,
      reason: 'QPS限流',
      retryAfter: waitTime,
    };
  }

  // 消耗令牌
  bucket.tokens -= 1;
  this.tokenBuckets.set(ruleId, bucket);

  return { allowed: true };
}
```

## 🎯 限流级别说明

### 1. 全局限流 (GLOBAL)

- **target**: `global`
- **作用范围**: 整个系统的所有请求
- **优先级**: 最高

### 2. 应用级限流 (APP)

- **target**: 应用标识 (appCode)
- **作用范围**: 特定应用的所有请求
- **优先级**: 第二

### 3. 接口级限流 (INTERFACE)

- **target**: 接口路径 (如 `/api/ai/invoke`)
- **作用范围**: 特定接口的所有请求
- **优先级**: 第三

### 4. 模型级限流 (MODEL)

- **target**: 模型ID
- **作用范围**: 特定模型的所有调用
- **优先级**: 第四

## 📊 限流策略

### 1. QPS限流（令牌桶算法）

- **优点**: 允许突发流量，平滑限流
- **参数**: 
  - `qpsLimit`: 每秒请求数
  - `burstSize`: 突发流量大小

### 2. 并发限流

- **优点**: 控制同时处理的请求数
- **参数**: `concurrentLimit`

### 3. 每日限额

- **优点**: 控制每日总调用量
- **参数**: `dailyLimit`
