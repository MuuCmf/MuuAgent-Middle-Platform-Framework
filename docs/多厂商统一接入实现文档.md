# 多厂商统一接入实现文档

## 📋 功能概述

后端服务已经实现了完整的多厂商统一接入功能，包括：

✅ **统一请求入参** - 屏蔽不同厂商的请求格式差异
✅ **统一出参格式** - 统一响应格式，屏蔽厂商差异
✅ **统一错误码** - 统一错误码定义和处理
✅ **厂商适配器** - 支持主流AI厂商的自动适配

private buildRequestData(model: Model, dto: AiInvokeDto, stream = false) {
  const provider = model.provider.toLowerCase();
  
  // Ollama 格式
  if (provider === 'ollama') {
    return {
      model: model.code,
      messages: dto.messages,
      stream,
      options: {
        temperature: dto.temperature,
        num_predict: dto.maxTokens,
      }
    };
  }
  
  // 智谱AI 格式
  if (provider === 'zhipu') {
    return {
      model: model.code,
      messages: dto.messages,
      temperature: dto.temperature,
      max_tokens: dto.maxTokens,
      stream,
    };
  }
  
  // DeepSeek 格式
  if (provider === 'deepseek') {
    return {
      model: model.code,
      messages: dto.messages,
      temperature: dto.temperature,
      max_tokens: dto.maxTokens,
      stream,
    };
  }
  
  // OpenAI 兼容格式 (OpenAI, Azure, 阿里云, 腾讯, 百度等)
  return {
    model: model.code,
    messages: dto.messages,
    temperature: dto.temperature,
    max_tokens: dto.maxTokens,
    stream,
  };
}

#### 响应格式转换器
位置：[response.transformer.ts](../service/src/ai/transformers/response.transformer.ts)

```typescript
@Injectable()
export class ResponseTransformer {
  /**
   * 转换AI响应为统一格式
   */
  transformAiResponse(provider: string, model: Model, response: any): UnifiedAiResponse {
    switch (provider.toLowerCase()) {
      case 'ollama':
        return this.transformOllamaResponse(model, response);
      case 'zhipu':
        return this.transformZhipuResponse(model, response);
      case 'deepseek':
        return this.transformDeepSeekResponse(model, response);
      default:
        return this.transformOpenAICompatibleResponse(model, response);
    }
  }
  
  /**
   * Ollama响应转换
   */
  private transformOllamaResponse(model: Model, response: any): UnifiedAiResponse {
    return {
      id: `ollama-${Date.now()}`,
      object: 'chat.completion',
      created: Math.floor(Date.now() / 1000),
      model: model.code,
      choices: [{
        index: 0,
        message: {
          role: 'assistant',
          content: response.message?.content || response.response || '',
        },
        finish_reason: response.done ? 'stop' : null,
      }],
      usage: response.eval_count ? {
        prompt_tokens: response.prompt_eval_count || 0,
        completion_tokens: response.eval_count || 0,
        total_tokens: (response.prompt_eval_count || 0) + (response.eval_count || 0),
      } : undefined,
      provider: 'ollama',
    };
  }
}
```

#### 错误码转换器
位置：[error.transformer.ts](../service/src/ai/transformers/error.transformer.ts)

```typescript
@Injectable()
export class ErrorTransformer {
  /**
   * 转换错误为统一格式
   */
  transformError(provider: string, error: any): UnifiedErrorResponse {
    const statusCode = error.response?.status;
    let unifiedCode = UnifiedErrorCode.MODEL_ERROR;
    
    // 根据HTTP状态码映射
    if (statusCode === 401) {
      unifiedCode = UnifiedErrorCode.INVALID_API_KEY;
    } else if (statusCode === 429) {
      unifiedCode = UnifiedErrorCode.RATE_LIMIT_EXCEEDED;
    } else if (statusCode === 404) {
      unifiedCode = UnifiedErrorCode.MODEL_NOT_FOUND;
    }
    
    // 提供商特定错误码映射
    const mapping = ProviderErrorCodeMapping[provider.toLowerCase()];
    if (mapping) {
      const providerCode = error.response?.data?.error?.code;
      unifiedCode = mapping[providerCode] || unifiedCode;
    }
    
    return {
      code: unifiedCode,
      message: getErrorMessage(unifiedCode),
      details: error.message,
      providerCode: error.response?.data?.error?.code,
      providerMessage: error.response?.data?.error?.message,
      retryable: isRetryableError(unifiedCode),
    };
  }
}
```

## 🔧 核心优势

### 1. **屏蔽厂商差异**
- 开发者无需关心不同厂商的API差异
- 统一的请求和响应格式
- 一次开发，多厂商兼容

### 2. **易于扩展**
- 新增厂商只需添加适配器
- 不影响现有代码
- 支持自定义扩展

### 3. **统一错误处理**
- 统一的错误码定义
- 自动错误码映射
- 友好的错误提示

### 4. **完整的日志记录**
- 记录原始请求和响应
- 记录厂商信息
- 便于问题排查
